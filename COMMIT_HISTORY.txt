================================================================================
COMMIT PROTOCOL (FOR ALL FUTURE COMMITS)
================================================================================

All commits must follow this format:

  "YYYY-MM-DD: [filename or change description] — [one-line context note]"

Examples:
  "2026-04-16: README.md — precision engineering framing, IP notice, travel notice"
  "2026-04-16: PRIMER_FOR_CLAUDE_CODE_20260416.txt — full project context primer"
  "2026-05-10: smnnip_lagrangian_tf.py — L2/L3 Noether gradient fix via TF autodiff"

The commit message is the Arrow of Time for the codebase. It should tell a
reader six months from now what changed and why — not just what the file is.

For multi-file pushes, commit each logically related group separately with
distinct messages rather than batching everything into one commit.
