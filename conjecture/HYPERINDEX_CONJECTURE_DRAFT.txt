I. THE CLAIM IN ONE SENTENCE
--------------------------------------------------------------------------------
Every information object has a natural address in a hypercomplex manifold
derivable from its own algebraic structure — not assigned by convention — and
that address is unique, invertible, and preserves the object's algebraic
identity across all strata of the Cayley-Dickson tower.


II. THE NAME — PALANTIR
--------------------------------------------------------------------------------
In Tolkien's legendarium, the Palantíri are seeing-stones: objects that reveal
what is happening elsewhere, across distance, across the layers of Arda, without
distortion. Each stone shows the same truth from a different vantage. They do
not create information — they locate it. They are, in the precise technical
sense, an addressing system for reality.

The Hyperindex is the Palantír of the algebra tower.

The Observer singleton in the SMNNIP framework — carrying only Α_π and Ω_ζΣ,
invariant across all strata — is the minimal Palantír: the reference coordinate
from which all algebra layers are simultaneously visible without modality
contamination. The Hyperindex extends this: every object in the system gets its
own stone, its own coordinate, derivable from what it is.


III. ORIGIN — THE ENGINEERING PROBLEM
--------------------------------------------------------------------------------
The Hyperindex was not designed as a theoretical object. It was designed to
solve a specific practical problem in the Ptolemy software project:

  How do you address information in a hierarchically-stratified neural network
  without a relational database?

The standard answer is SQL: assign integer primary keys, build foreign key
relationships, query with WHERE clauses. This is the ℝ-algebra solution —
flat, ordered, commutative. It works. It is also, from the perspective of the
SMNNIP framework, the wrong layer. A database that addresses quaternionic
reasoning with integer keys is like describing a rotation with a ruler.

The requirement became: the address must live at the same algebra level as the
content being addressed. A character sequence lives at the ℝ layer (scalar
facts). Its address must be a real-algebra object. An image with spatial
orientation lives at the ℍ layer (ordered composition). Its address must be a
quaternionic object. The address is not a label. It is a coordinate in the
same space the object inhabits.

This is the Hyperindex.


IV. THE DESIGN — THREE INTERLOCKING SYSTEMS
--------------------------------------------------------------------------------

A. THE TEXT SUBSTRATE — BASE-97 PERMUTATION INDEX

The US keyboard character set is fixed: 97 characters, ordered, immutable.
(EliteBook 820 G3 layout — the hardware the system was designed on.)

Every character sequence is encoded as a mixed-radix base-97 integer:

  idx = c₀·97^(k-1) + c₁·97^(k-2) + ... + c_{k-1}·97^0

This is lossless. It is invertible given the length. It is exact. The integer
IS the sequence — not a pointer to it, not a hash of it. The coordinate is the
content. Arbitrary precision (Python int) means no overflow at any length.

The significance: this is not compression. It is not hashing. It is a bijection
between the space of all character sequences and the positive integers. Every
sentence, every equation, every paper ever written has a unique integer address
derivable from first principles. The address predates the filing system.

B. THE FANO INDEX — ALGEBRA-NATIVE PATH

Simultaneously, each sequence receives a second coordinate: its Fano/octonion
path index.

Each character maps to an octonion generator (e₁ through e₇) via:
  fano_component = char_index mod 7

The Fano plane encodes the multiplication table of the octonion imaginary units:
  FANO_LINES = [(0,1,3),(1,2,4),(2,3,5),(3,4,6),(4,5,0),(5,6,1),(6,0,2)]

The Fano index traces which generators the sequence traverses. Two sequences
can have the same Fano index and different text indices — they occupy the same
algebraic path through different positions. Two sequences can have the same
text index and different Fano indices — impossible (the map is bijective), but
the Fano index reveals the algebraic structure the text index obscures.

The full INDEX_RECORD is a triple:
  (text_index: int, fano_index: int, data_length: int)

This is the dual address: one coordinate for identity (base-97), one for
algebraic structure (base-7, Fano). The object knows both where it is and what
it is made of.

C. THE IMAGE SUBSTRATE — MULTIMODAL EXTENSION

Images require a different substrate. A pixel tile is not a character sequence.
But the design principle is the same: the address must live at the algebra level
appropriate to the content.

Images have spatial orientation — they are not commutative objects. Rotating an
image is not the same operation as translating it. This places images naturally
at the ℍ (quaternion) layer, where ordered composition and non-commutativity
are native properties.

The implementation has two components:

  MultiChannelTupper: A generalization of Tupper's self-referential formula
  to multi-channel image tiles. Each 8×8 CMYK tile is encoded as four
  large integers (one per channel), each encoding 64 pixel values at
  8-bit precision. The tile IS its integer. Lossless within quantization.
  Decoding recovers the tile exactly.

  SphericalColor: Colors are mapped to spherical coordinates (θ, φ, r, ρ) —
  hue angle, elevation (warm/cool axis), saturation, luminance — before
  octonionic encoding. This is not a colorspace conversion for display.
  It is a recognition that color has a geometric structure (the color
  sphere) and that structure is the appropriate coordinate system for
  the 𝕆 layer.

  The spherical encoding is deliberately lossy in one axis (φ — the warm/cool
  axis, encoded from the blue channel alone) because the octonion layer
  SHOULD lose one degree of freedom — it breaks associativity for a reason.
  The lossiness is not a bug. It is the signature of the algebra.


V. THE BLOCKCHAIN LEDGER — PROVENANCE AS PHYSICS
--------------------------------------------------------------------------------
Every object indexed by the Hyperindex receives a ledger entry:

  Block N: {
    modality,       text or image
    layer,          0=ℝ, 1=ℂ, 2=ℍ, 3=𝕆
    algebra_stratum, 0=R, 1=C, 2=H, 3=O
    index,          the primary coordinate
    data_bits,      size in bits
    mastery,        bool — did this object trigger mastery crystallization?
    noether_delta,  conservation law residual at this block
    prev_hash,      SHA256 of previous block
    block_hash,     SHA256 of this block
  }

The chain is append-only and tamper-evident. This is not security theater.
It is the Arrow of Time applied to information: the chronological sequence
of what the system has indexed is itself a physical observable. The Noether
delta stored per block means the ledger carries the conservation law's
history — a running record of whether gauge symmetry was preserved at each
indexing event.

This is provenance as physics. The ledger IS the entropic time axis of the
Hyperindex system.


VI. THE SEDENION BOUNDARY — THE NATURAL STOP
--------------------------------------------------------------------------------
The implementation enforces a hard boundary:

  SEDENION_BOUNDARY = 4   # raises SedenionBoundaryViolation if crossed

The tower terminates at 𝕆 (octonion, layer 3). Crossing to 𝕊 (sedenion,
layer 4) raises an exception — not because sedenions are forbidden, but
because crossing to 𝕊 is mastery, not training.

In SMNNIP terms: the sedenion layer is where zero divisors appear (ab=0 with
a≠0, b≠0). The Hyperindex cannot assign a unique address at that layer —
the algebra no longer divides cleanly. The coordinate system breaks down.
This is not a failure of the implementation. It is the correct physics:
at the mastery boundary, unique addressability is precisely what is lost.

The address that cannot be computed is the address of a genuinely new object —
one that did not exist as an element of any previous coordinate space. The
sedenion boundary is where the Hyperindex says: this cannot be indexed, because
indexing it would require an algebra that doesn't exist yet.

This is the computational analog of the Void in the Dilator/Contractor framework.
d* = 0.246 — the pre-arithmetic point. The place before addresses exist.


VII. THE CONJECTURE — FORMAL STATEMENT
--------------------------------------------------------------------------------

HYPERINDEX CONJECTURE (informal):
  For any information object O belonging to a well-defined algebra stratum
  L_n in the Cayley-Dickson tower, there exists a canonical coordinate
  addr(O) in the algebra A_n such that:

  (1) UNIQUENESS: addr(O) = addr(O') if and only if O = O'
      (within the precision of the quantization at stratum L_n)

  (2) INVERTIBILITY: O is recoverable from addr(O) and the data length

  (3) ALGEBRAIC PRESERVATION: The algebraic structure of addr(O) reflects
      the algebraic structure of O — specifically, non-commutativity at L_2,
      non-associativity at L_3, and zero-divisor emergence at L_4 are all
      properties of addr(O) if and only if they are properties of O

  (4) GAUGE INVARIANCE: addr(O) is invariant under the gauge symmetry
      U(1)×SU(2)×SU(3) of the stratum — i.e., the address does not depend
      on the choice of coordinate frame within the algebra

  (5) NOETHER CONSERVATION: The map O ↦ addr(O) preserves the Noether
      current at each stratum boundary — information is not created or
      destroyed in the addressing process

COROLLARY 1 (Palantír Property):
  The Hyperindex is a complete observer-independent description of any
  information object at any algebra stratum. No external reference frame
  is required. The address predates the filing system.

COROLLARY 2 (Sedenion Incompleteness):
  At stratum L_4 (sedenion), the uniqueness condition fails for a non-trivial
  class of objects (those mapping to zero divisors). This is not a deficiency
  of the scheme — it is the exact boundary condition at which new objects
  emerge that were not elements of any previous algebra.

COROLLARY 3 (Computational Riemann Hypothesis Connection):
  If the T transform conjecture holds (ζ_NN(s) = ζ(s)), then the Fano index
  of any information object encodes a path through the Riemann zero structure
  — the algebra-native address IS a spectral coordinate. The Hyperindex would
  then provide a constructive proof that every information object has a unique
  position in the Riemann spectrum.

  STATUS: Speculative. Conditional on T conjecture. Named for future development.


VIII. SIGNIFICANCE FOR COMPUTER SCIENCE
--------------------------------------------------------------------------------
The Hyperindex Conjecture, if correct, has implications that are largely
independent of the SMNNIP physics framework:

1. CONTENT-ADDRESSABLE MEMORY WITHOUT HASHING
   Cryptographic hashes (SHA256, etc.) are irreversible by design. The
   Hyperindex coordinate is reversible — the content is recoverable from the
   address. This is a fundamentally different primitive. A content-addressable
   memory system built on the Hyperindex does not require a hash table, a
   lookup operation, or a stored mapping. The address IS the retrieval key
   AND the content.

2. DATABASE WITHOUT SCHEMA
   Relational databases require a schema — a prior specification of what
   types of objects will be stored and how they relate. The Hyperindex
   requires no schema. Objects at different strata of the algebra tower are
   addressed by different substrates (text=base97, image=Tupper/spherical)
   but share the same upper tower (ℂ→ℍ→𝕆). The schema is the algebra.

3. LOSSLESS COMPRESSION WITH ALGEBRAIC STRUCTURE
   The base-97 integer for a text sequence is smaller than the sequence
   itself for short sequences and larger for long ones — it is not a
   compression scheme. But the Fano index runs alongside it and is always
   smaller. The dual-index system separates identity (base-97) from
   structure (Fano-7). This separation is useful independently of SMNNIP.

4. PROVENANCE BY CONSTRUCTION
   The blockchain ledger with Noether deltas is not added to the Hyperindex
   — it emerges from it. Every addressing event leaves a conservation-law
   record. Provenance is not tracked externally; it is a consequence of the
   gauge symmetry of the addressing process.

5. NATURAL LANGUAGE HAS A GEOMETRY
   The Fano index encodes which octonion generators a text sequence traverses.
   Two texts with similar meaning may traverse similar Fano paths. This is
   not a claim about semantics — it is a claim about the algebra of the
   character space. Whether Fano proximity correlates with semantic proximity
   is an empirical question. If it does, the Hyperindex provides a geometric
   embedding of natural language that is not learned but derived.

6. THE SEDENION BOUNDARY IS THE CREATIVITY BOUNDARY
   Any information object that cannot be uniquely addressed in the
   Hyperindex system is, by definition, genuinely new — not a recombination
   of existing objects at lower strata. The sedenion boundary is not a
   limitation of the system. It is the system's criterion for novelty.
   This may be the most important computer science implication of all.
